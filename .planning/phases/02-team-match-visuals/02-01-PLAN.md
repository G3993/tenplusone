    Extract grid.html (~800 lines) into 5 TypeScript modules:

    **FluidSimulation.ts** -- Self-contained class encapsulating the entire fluid sim from grid.html lines 131-453.
    - Constructor takes a canvas element and optional config (TEXTURE_DOWNSAMPLE=1, DENSITY_DISSIPATION=0.98, VELOCITY_DISSIPATION=0.98, PRESSURE_DISSIPATION=0.8, PRESSURE_ITERATIONS=25, CURL=30, SPLAT_RADIUS=0.005)
    - `getFluidWebGLContext()`, `initFluidFramebuffers()`, `createFluidFBO()`, `createFluidDoubleFBO()` become private methods
    - All shader sources (baseVS, clearFS, displayFS, splatFS, advectionFS, divergenceFS, curlFS, vorticityFS, pressureFS, gradSubFS) become private class properties compiled in constructor
    - `fluidStep()` becomes public `step()` method
    - `fluidSplat()` becomes public `splat(x, y, dx, dy, color)` method
    - `fluidMultipleSplats()` becomes public `multipleSplats(amount)` method
    - Mouse/touch pointer state management stays internal (array of pointer objects with id, x, y, dx, dy, down, moved, color)
    - `getNextColor()` returns `[base, base, base]` where base = 0.6 + Math.random() * 0.4
    - `dispose()` method: delete all textures, framebuffers, programs, lose WebGL context
    - `attachPointerEvents(element: HTMLElement)` method: attach mousemove/mousedown/mouseup/touchstart/touchmove/touchend handlers to given element, return cleanup function
    - `resizeCanvas()` becomes public, called externally on window resize

    **GridMesh.ts** -- InstancedMesh-based grid management replacing the 1024 individual mesh approach.
    - Constructor takes `scene: THREE.Scene`, optional config `{ gridSize: 32, cellSize: 1.3, gap: 0.02 }`
    - Use TWO `THREE.InstancedMesh` instances: one with white MeshStandardMaterial (metalness=0.1, roughness=0.3), one kept for future use
    - Actually, simpler approach: single InstancedMesh with 1024 instances. Use per-instance visibility by setting scale to 0 for hidden cells and normal scale for visible cells.
    - `setMatrixAt` for positioning in constructor: `x = i * (cellSize + gap) - offset`, `z = j * (cellSize + gap) - offset`, `y = cellSize / 2`
    - Public `setCellVisible(index: number, visible: boolean)` -- sets scale to (1,1,1) or (0,0,0) via matrix
    - Public `setGeometry(geometry: THREE.BufferGeometry)` -- swap between cube and sphere geometries (creates new InstancedMesh, disposes old)
    - Store cell states in `Uint8Array(1024)` for fast lookup
    - `dispose()` method: dispose geometries, materials, remove from scene
    - Create collision plane for raycasting (invisible PlaneGeometry at y=0, receives shadow)
    - Shadow-receiving plane: `new THREE.Mesh(new THREE.PlaneGeometry(200,200), new THREE.ShadowMaterial({opacity: 0.35}))`, rotation.x = -PI/2, position.y = -1.0
    - Box geometry: `new THREE.BoxGeometry(cellSize, cellSize, cellSize)`
    - Sphere geometry: `new THREE.SphereGeometry(cellSize * 0.55, 24, 24)`

    **transitions.ts** -- Pure functions for animation delay calculations.
    - Export `type TransitionType = 'wave' | 'spiral' | 'random' | 'rows' | 'columns' | 'diagonal'`
    - Export `function getDelay(pixelId: number, type: TransitionType, gridSize: number = 32): number`
    - Exact delay formulas from grid.html lines 574-583:
      - wave: `Math.sqrt((row-cr)**2 + (col-cr)**2) * 25`
      - spiral: `(Math.atan2(row-cr, col-cr) + Math.PI) * 50 + Math.sqrt((row-cr)**2 + (col-cr)**2) * 30`
      - random: `Math.random() * 800`
      - rows: `row * 60 + Math.random() * 20`
      - columns: `col * 60 + Math.random() * 20`
      - diagonal: `(row + col) * 30`
    - Export `const TRANSITIONS: TransitionType[]` = all 6 types

    **PixelGridController.ts** -- Main orchestrator class.
    - Constructor takes `container: HTMLElement` and optional `GridConfig` (extends grid mesh config + camera/lighting settings)
    - Creates Three.js scene (background = null for transparency), PerspectiveCamera(45, aspect, 0.1, 1000), WebGLRenderer(antialias=true, alpha=true)
    - Camera position: `(0, 80, 0)` looking at `(0, 0, 0)`
    - Renderer: `setPixelRatio(Math.min(devicePixelRatio, 2))`, `setClearColor(0x000000, 0)`, `shadowMap.enabled = true`, `shadowMap.type = THREE.VSMShadowMap`
    - Lighting setup from grid.html lines 477-502: AmbientLight(0xffffff, 0.25), DirectionalLight(0xffffff, 0.9) at (15,50,15) with 2048x2048 shadow map and VSM settings, rim light DirectionalLight(0x4488ff, 0.3) at (-50,20,-50), two fill lights
    - Creates FluidSimulation on a separate canvas element (appended to container before the Three.js canvas, z-index 0 vs 1)
    - Creates GridMesh
    - `showLogo(pixels: number[], transition: TransitionType)` -- iterates 1-1024, compares desired state vs current, schedules flips via setTimeout with getDelay
    - `flipCell(index: number, toVisible: boolean)` -- animates a single cell: sets GridMesh cell visibility, runs a 400ms ease-out-cubic animation on the group's rotation.x and position.y bounce (from grid.html lines 659-698)
    - `startMatchMode(homePixels: number[], awayPixels: number[], switchInterval: number = 4000)` -- alternates between two logos on a timer
    - `stopMatchMode()` -- clears the interval
    - `setScale(scale: number)` -- sets gridGroup scale (0.3-1.5 range)
    - `setRotation(degrees: number)` -- sets gridGroup rotation.y
    - `toggleShape()` -- switches between cube and sphere geometry
    - `toggleDarkMode()` -- inverts all cell states with diagonal animation
    - `setZoom(height: number)` -- sets camera.position.y (40-150 range), called from wheel handler
    - Animation loop: `requestAnimationFrame`, calls `fluidSim.step()` then `renderer.render(scene, camera)`
    - `resize()` method for window resize: update camera aspect, renderer size, fluid canvas size
    - `dispose()` -- cancel animation frame, clear intervals, dispose renderer/fluidSim/gridMesh, remove canvases from container
    - Wheel handler on renderer.domElement: zoom camera position.y between 40-150

    **PixelGrid.tsx** -- React wrapper.
    ```typescript
    interface PixelGridProps {
      logoPixels: number[];
      matchMode?: { awayPixels: number[]; switchInterval?: number; };
      height?: string; // CSS height, default '60vh'
      className?: string;
    }
    ```
    - `useRef<HTMLDivElement>` for container, `useRef<PixelGridController>` for controller
    - `useEffect` with empty deps: create controller, guard against React 18 strict mode double-mount with `if (controllerRef.current) return;`
    - Cleanup: call `controller.dispose()`, set ref to null
    - Second `useEffect` watching `[logoPixels]`: call `controllerRef.current?.showLogo(logoPixels, 'wave')`
    - Third `useEffect` watching `[matchMode]`: if matchMode, call startMatchMode; else stopMatchMode
    - Window resize handler in main useEffect
    - Render: `<div ref={containerRef} className={styles.container} style={{ height: height || '60vh' }} />`

    **PixelGrid.module.css**:
    ```css
    .container { width: 100%; position: relative; overflow: hidden; cursor: crosshair; }
    ```

    IMPORTANT: Do NOT use React state for any per-frame values. All Three.js state lives in the controller class.
    IMPORTANT: The fluid sim canvas and Three.js canvas are TWO separate canvases layered via CSS (position: absolute, z-index 0 and 1) inside the container div. This matches the prototype's approach.
    IMPORTANT: For the flip animation, do NOT clone materials per flip like the prototype does. With InstancedMesh, visibility is controlled by scale (0,0,0 vs 1,1,1) set via setMatrixAt. The animation is purely the group rotation.x and position.y bounce -- but since we use InstancedMesh, the flip animation needs to be handled differently. Use a per-cell animation state array in the controller, and in the animation loop, update matrices for any currently-animating cells.
